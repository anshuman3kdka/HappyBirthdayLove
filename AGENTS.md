# Nemo System Instructions

## Documentation Conventions
You must maintain the system documentation (`README.md`) constantly. Whenever a new feature is added, or an architectural / design decision is made during the session, update the `README.md` file to reflect these changes.
Keep the tone cinematic and emphasize the digital birthday experience objective.

When making updates:
- Add a new bullet point to the `📝 Update Log & Future Enhancements` section at the bottom of the `README.md`.
- Enhance the `Core Features` or `Design Philosophy` sections with the new insights or features recently requested.
